# Layered desktop architecture and async load jobs

The desktop path is standardized on egui with a strict three-layer split (Desktop UI Layer, Application Layer, Retained Data and Index layers) under a single desktop module boundary, and the legacy iced path is removed. UI-to-app interaction uses an Intent/Event Boundary, loading runs as worker-thread Load Jobs, and results are applied with Load Generation latest-wins gating so stale completions are ignored while keeping Structured Filter then Rendered Text Search as an explicit Query Pipeline. We chose this over a looser split or synchronous loading because it is more maintainable and testable, keeps benchmark APIs stable through module re-exports, and preserves deterministic selection behavior through Selection Continuity rules.
